# PRODUCT.md — 햄버거 가게 키오스크 시스템

---

## 1. 제품 목적

본 제품은 햄버거 가게의 주문 프로세스를 디지털화하여, 고객이 직원의 도움 없이 스스로 메뉴를 탐색하고 주문·결제까지 완료할 수 있도록 지원하는 셀프 키오스크 시스템입니다.

- 주문 대기 시간 단축 및 매장 운영 효율 향상
- 고객 경험 개선을 통한 재방문율 제고
- 관리자가 메뉴·주문·매출을 통합 관리할 수 있는 백오피스 기능 제공

---

## 2. 대상 사용자

| 구분 | 설명 |
|------|------|
| **일반 고객 (사용자)** | 매장을 방문하여 키오스크 화면에서 직접 메뉴를 선택하고 주문하는 모든 연령층의 고객 |
| **매장 관리자** | 메뉴 등록·수정·삭제, 주문 현황 확인, 매출 분석 등 운영 전반을 담당하는 매장 직원 또는 점주 |

---

## 3. 주요 기능

### 3.1 사용자 기능

| 기능 | 설명 |
|------|------|
| **메뉴 목록 조회** | 카테고리(버거, 사이드, 음료 등)별로 전체 메뉴와 가격·이미지를 확인 |
| **장바구니 담기** | 원하는 메뉴를 수량 지정하여 장바구니에 추가, 수정 및 삭제 가능 |
| **주문하기** | 장바구니 내역을 최종 확인 후 결제 진행 및 주문 번호 발급 |

### 3.2 관리자 기능

| 기능 | 설명 |
|------|------|
| **메뉴 관리** | 메뉴 항목 추가·수정·삭제, 카테고리 및 품절 상태 관리 |
| **주문 기록 관리** | 전체 주문 내역 조회 및 주문 상태(접수·완료·취소) 변경 |
| **매출 관리** | 일별·월별 매출 집계 및 인기 메뉴 통계 조회 |

---

## 4. 핵심 제약사항

### 4.1 기능적 제약

- 고객 주문은 장바구니에 최소 1개 이상의 메뉴가 담긴 경우에만 진행 가능
- 품절 처리된 메뉴는 장바구니에 추가할 수 없음
- 주문 완료 후에는 고객이 직접 취소·변경 불가 (관리자만 처리 가능)
- 관리자 기능은 인증(로그인)된 관리자만 접근 가능

### 4.2 비기능적 제약

- 키오스크 화면은 터치 입력에 최적화된 UI 제공 (최소 터치 영역 44×44px 이상)
- 메뉴 목록 조회 응답 시간은 3초 이내 보장
- 시스템은 매장 운영 시간(07:00–22:00) 동안 99% 이상의 가용성 유지
- 결제 처리 및 주문 데이터는 안전하게 저장되어야 하며, 고객 개인정보는 최소한으로 수집

### 4.3 범위 외 (Out of Scope)

- 온라인(앱/웹) 주문 및 배달 기능
- 멤버십·포인트 적립 시스템
- 외부 POS 연동 (1차 릴리즈 기준)

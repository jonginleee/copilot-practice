# 3-2. GitHub Copilot으로 단위 테스트 작성하기

## 실습 범위: 테스트 작성 ~ 디버깅 팁

3-1에서 작성한 FastAPI 회원가입 코드를 **테스트로 검증**하고, 다음을 학습합니다:
- 테스트 작성 (Step 1): Copilot으로 빠르게 테스트 케이스 생성
- Test Explorer (Step 2): VS Code 네이티브 도구로 시각적 관리
- 커버리지 분석 (Step 3): 단순히 테스트 통과가 아니라 **품질 있는 테스트** 작성
- 엣지 케이스 (Step 4): 실제 버그가 잘 걸리는 시나리오
- 디버깅 팁 (Step 5): Copilot과 Test Explorer 통합으로 실패 진단

---

## Step 1: 테스트 환경 설정

1. `/setupTests`를 입력해 테스트 환경을 설정합니다.

2. 테스트 프레임워크 선택 화면이 나올 경우 `pytest`를 선택합니다.

3. 이후 나오는 메시지에 따라 `Apply Changes`, `Configure Test Framework`를 클릭해 설정한다.

4. VS Code 왼쪽 사이드바에서 `Testing`을 클릭한다.

5. 테스트 탐색기에서 테스트를 실행해보고 테스트가 제대로 실행되는 지 확인한다.

6. 제대로 되지 않는 경우 Copilot으로 수정해서 테스트가 작동하도록 한다.

## Step 2: Copilot으로 테스트 작성하기

1. Chat에서 아래 프롬프트를 입력:

```
#auth.py의 POST /auth/register를 테스트하는 pytest 코드를 작성해줄래?

테스트 케이스:
- 성공: (email, password) 입력 -> 201, user 반환
- 실패: 이메일 형식 오류 -> 400 INVALID_EMAIL
- 실패: 비밀번호 약함 -> 400 WEAK_PASSWORD
- 실패: 중복 이메일 -> 409 DUPLICATE_EMAIL

```

2. 테스트 탐색기를 열어봅니다.

3. 테스트 자동 발견되는 것을 확인합니다.

Test Explorer에서:
- `tests/` 폴더 아래 모든 `test_*.py`와 `*_test.py` 파일 자동 발견
- 각 테스트 함수가 트리 구조로 표시됨
- 테스트가 추가되면 "새로고침" 버튼으로 수동 갱신

4. 테스트 실행

**전체 테스트 실행:**
- Test Explorer 최상단의 ▶️ (재생) 버튼 클릭

**특정 테스트만 실행:**
- 테스트 함수 이름 옆 ▶️ 클릭
- 예: `test_register_success` 옆의 ▶️ 클릭


5. 테스트 결과 확인

**인라인 결과:**
- ✓ (녹색): 테스트 통과
- ✕ (빨간색): 테스트 실패
- ⊗ (회색): 아직 실행되지 않음

6. 실패한 테스트에서 `Fix Test Failure`를 클릭합니다.

![alt text](image-1.png)

7. 작동하지 않는 경우 프롬프트에 다음을 입력합니다.

```
#sym:test_register_success 테스트가 실패하는 데 고쳐줘.

```

8. 테스트가 작동하지 않는 경우 프롬프트에 다시 지시를 해서, 테스트를 모두 작동하도록 합니다.

## Step 3: 다른 파일도 테스트 작성해보기

1. `test_security.py` 파일을 열어봅니다.

2. 프롬프트를 아래와 같이 입력합니다.

`/tests`

3. 작성된 테스트를 확인합니다.

4. 테스트를 코드로 쓰지 않을 경우 테스트를 써달라고 프롬프트에 지시합니다.

5. 테스트가 잘 작동하는 지 확인합니다.


## Step 4: 검증 요구사항을 더 추가합니다.

1. 아래와 같이 프롬프트를 입력합니다.

```
#tests/test_main.py의 테스트들을 보고,
아직 테스트하지 않은 엣지 케이스들을 추가해줄래?

추가할 케이스:
- 매우 긴 이메일 (200자)로 가입 시도 -> 400 INVALID_EMAIL
- 공백만 있는 비밀번호 -> 400 WEAK_PASSWORD
- 유효한 이메일이지만 특수문자 없는 비밀번호 -> 400 WEAK_PASSWORD
- 정확히 8자(최소 길이)인 유효한 비밀번호 -> 201 성공
```

2. 추가된 테스트를 확인하고, 테스트가 제대로 되는 지 확인한다.

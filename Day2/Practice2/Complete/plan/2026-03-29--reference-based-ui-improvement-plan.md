# 구현 계획: 레퍼런스 기반 키오스크 UI 개선
첨부된 레퍼런스 화면을 참고하여 키오스크 프론트 UI를 개선한다. 제품 범위(오프라인 키오스크)와 기존 데이터 흐름(메뉴 조회 -> 장바구니 -> 주문 생성 -> 저장/응답)은 유지하며, 주문 기능 동작을 해치지 않는 선에서 정보 구조와 시각 완성도를 높인다.

## 아키텍처 및 설계
- 적용 대상
  - 메인 주문 화면: [src/frontend/kiosk_app.py](src/frontend/kiosk_app.py)
  - 메뉴 페이지(보조): [src/frontend/pages/menu.py](src/frontend/pages/menu.py)
  - API 호출 계층(동작 유지 검증): [src/frontend/api_client.py](src/frontend/api_client.py)
- 설계 방향
  - 레퍼런스의 시각 구조를 반영해 상단 네비게이션, 좌측 카테고리 가이드, 카드형 메뉴 리스트를 재구성한다.
  - 메뉴 카드에는 이미지, 메뉴명, 설명, 가격, 핵심 CTA를 일관된 계층으로 배치한다.
  - 장바구니/주문 영역은 사용자가 현재 선택 상태와 결제 직전 액션을 빠르게 파악하도록 고정 또는 시인성 높은 레이아웃으로 유지한다.
  - 색상 체계는 레퍼런스의 컬러 톤을 기준으로 정의하되, 프로젝트 내 구현 가능한 스타일 토큰으로 치환해 일관 적용한다.
- 제약 및 호환성
  - Streamlit 기반 구현을 유지하고, backend API 계약은 변경하지 않는다.
  - 터치 입력 최소 크기(44x44px 이상)와 가독성(텍스트 대비/간격)을 충족한다.
  - 품절 메뉴 비활성화, 빈 장바구니 주문 제한 등 기존 도메인 규칙을 UI에서 동일하게 유지한다.
  - 아이콘/폰트/이미지는 외부 자산 직접 복제 없이 프로젝트 내 대체 자산을 사용한다.
- 검증 기준
  - UI 개선 후에도 주문 플로우(담기 -> 수량 변경 -> 주문하기 -> 완료 표시)가 동일하게 동작해야 한다.
  - 모바일 폭(소형 태블릿 포함)과 데스크톱 폭에서 레이아웃 붕괴 없이 핵심 CTA가 노출되어야 한다.

## 작업 목록
- [x] 레퍼런스 화면 요소 분해 및 매핑
  - 수용 기준: 레퍼런스의 핵심 요소(내비게이션, 카테고리, 카드, CTA)를 현재 화면 컴포넌트에 1:1 또는 대체 매핑한 표가 작성된다.
- [x] 디자인 토큰 정의
  - 수용 기준: 색상, 타이포, 간격, 라운드, 버튼 상태를 Streamlit 스타일 기준으로 정의하고 재사용 가능한 규칙으로 문서화한다.
- [x] 메인 레이아웃 재구성
  - 수용 기준: 상단/좌측/콘텐츠/장바구니의 4영역 구조가 적용되고, 해상도별로 핵심 정보가 유지된다.
- [x] 메뉴 카드 UI 개선
  - 수용 기준: 카드 내 정보 위계(이미지, 이름, 설명, 가격, 버튼)가 일관되며 품절 상태 표시가 명확하다.
- [x] 주문 인터랙션 시각 피드백 정비
  - 수용 기준: 장바구니 담기, 수량 수정, 삭제, 주문 완료에 대해 성공/실패 메시지가 레이아웃을 깨지 않고 일관되게 표시된다.
- [x] 반응형/터치 사용성 점검
  - 수용 기준: 주요 버튼/입력 컨트롤이 터치 기준을 만족하고 작은 화면에서도 조작 가능하다.
- [x] 회귀 검증
  - 수용 기준: 기존 주문 기능 시나리오(정상/예외)와 관련 테스트가 통과한다.
- [x] 결과 기록
  - 수용 기준: 변경 포인트, 비교 스크린샷 항목, 확인된 제약을 계획 문서에 반영한다.

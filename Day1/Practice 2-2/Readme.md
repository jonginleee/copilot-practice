## 사전 준비

### 준비물
- VS Code 최신 버전
- GitHub Copilot + Copilot Chat 확장 설치
- GitHub 로그인 완료
- Chat view 단축키 확인: `Ctrl+Alt+I`
- Chat 상단에서 모드 전환과 모델 선택 UI가 보이는지 확인

### 이번 실습에서 집중할 Chat 기능
- 파일을 첨부하거나 열어 둔 파일을 바탕으로 질문하기
- Ask 모드와 Agent 모드의 차이 비교하기
- 같은 질문을 서로 다른 모델에 던져 응답 차이 비교하기
- 후속 질문으로 대화를 이어 가고, 새 대화로 맥락을 분리하기

## 실습 환경 설정

### 사용할 파일

실습에는 `Day1\Practice 2-2\Start\` 폴더의 다음 파일을 사용합니다.

- `user_manager.py`: 불완전한 사용자 관리 클래스
- `data_handler.py`: 데이터 처리 함수 모음

### 준비 작업
1. `user_manager.py`와 `data_handler.py`를 미리 열어 둡니다.
2. Chat view를 열고 Ask 모드와 Agent 모드가 모두 보이는지 확인합니다.
3. Chat 상단에서 모델 선택 드롭다운이 있는지 확인합니다.

## 기능별 실습

## 실습 1: 파일 첨부로 요구사항 기반 구현

### 학습 목표
- 요구사항 문서를 Chat에 첨부하는 방법 익히기
- 요구사항 문서와 코드 파일을 함께 컨텍스트로 제공하기
- 첨부된 문서를 근거로 더 정확한 구현 요청하기

### 실습 단계

#### 1단계: Chat 열기
1. `Ctrl+Alt+I`를 누르거나 활동 표시줄에서 Chat view를 엽니다.
2. 새 대화를 시작합니다.

#### 2단계: 요구사항 문서 만들기
1. `바탕화면에서 REQUIREMENTS.md` 파일을 만듭니다.
2. 아래 내용을 작성합니다.

```markdown
# UserManager 개선 요구사항

## 배경
기존 UserManager 클래스를 보안, 성능, 기능 측면에서 개선해야 함

## 요구사항

### 1. 검증 강화
- username: 3-20자, 알파벳과 숫자만 허용
- email: 유효한 이메일 형식 검증
- age: 18-120 범위 검증

### 2. remove_user 메서드 구현
- username으로 사용자 찾기
- 사용자 없으면 False 반환
- 사용자 있으면 제거 후 True 반환

### 3. 예외 처리 추가
- username이 이미 존재하면 DuplicateUserError
- 검증 실패하면 ValidationError

### 4. 성능 개선
- username으로 빠른 조회 가능하도록 dict 사용
```

#### 3단계: 파일 첨부 후 구현 요청
1. Chat 입력창에서 작성한 `REQUIREMENTS.md` 파일을 끌어다 놓습니다.
2. 모드를 `ASK`로 변경합니다.
2. 아래 프롬프트를 입력합니다.

```text
첨부한 요구사항 문서를 바탕으로 요구사항이 잘 구현되어 있어?
수정해야 할 부분을 핵심만 담아서 알려줘.
```

#### 4단계: 응답 검토

응답 내용을 검토합니다.

#### 5단계: 프로젝트의 Requirements로 이동

해당 `REQUIREMENTS.md` 파일을 프로젝트 폴더에 넣습니다.

#### 5단계: 후속 요구사항 추가

1. 모드를 `Agent`로 바꿉니다. 그리고 #을 입력해 파일을 추가해주고 아래와 같이 입력해줍니다.

```text
#file:requirements.md 
추가 요구사항이 있어.
모든 add_user와 remove_user 호출에 대해 간단한 로깅이 필요한데 요구사항 문서에 반영해줘.
```

2. Copilot이 문서를 잘 업데이트 해주는 지 확인합니다.

## 실습 2: Ask 모드와 Agent 모드 비교

### 학습 목표
- Ask 모드와 Agent 모드의 차이를 이해하기
- 같은 문제를 두 모드에 각각 맡겨 흐름 차이 비교하기
- 어떤 작업에 어떤 모드가 맞는지 기준 세우기

### 실습 단계

#### 1단계: Ask 모드에서 제안 받기
1. Chat 모드를 Ask로 설정합니다.
2. `user_manager.py`를 컨텍스트에 추가합니다.
3. 아래 프롬프트를 입력합니다.

```text
remove_user 메서드를 구현하려고 해.
username으로 사용자를 찾아 제거하고, 없으면 False를 반환하는 방식의 구현안을 제안해줘.
```

4. 진행하라고 한 후 실제 수정 작업까지 이뤄지는 지 확인합니다.

```
구현안대로 진행해줘.
```

#### 2단계: Agent 모드에서 동일 문제 다루기
1. Chat 모드를 Agent로 전환합니다.
2. `user_manager.py`와 `data_handler.py`를 컨텍스트에 추가합니다.
3. 아래 프롬프트를 입력합니다.

```text
#file:requirements.md를 바탕으로 부족한 부분을 분석하고, 분석된 내용을 구현해줘.
```

4. Agent가 분석, 계획, 수정 제안까지 더 길게 이어 가는지 관찰합니다.

## 실습 3: 모델 변경과 응답 품질 비교

### 학습 목표
- 모델별 응답 특성을 비교하기
- 같은 질문에 대한 응답 차이를 확인하기
- 작업 유형별 모델 선택 기준 세우기

### 실습 단계

#### 1단계: 현재 모델 확인
1. Chat 상단 모델 선택 드롭다운을 확인합니다.
2. `Other Models`에 `GPT-4o` 모델을 선택합니다.

#### 2단계: 동일 질문으로 두 모델 비교
1. 첫 번째 모델에서 `user_manager.py`를 컨텍스트에 추가합니다.
2. 아래 질문을 입력합니다.

```text
이 코드에서 보안 관점에서 문제가 될 수 있는 부분을 찾아줘.
특히 사용자 정보 검증과 접근 제어 관점에서 설명해줘.
```
3. 답변을 메모장에 기록합니다.

4. `/clear`를 입력해 대화창을 초기화 시킵니다.=
5. `Other Models`에 `GPT-5.4` 모델을 선택합니다.
6. 같은 질문을 입력합니다.
```text
이 코드에서 보안 관점에서 문제가 될 수 있는 부분을 찾아줘.
특히 사용자 정보 검증과 접근 제어 관점에서 설명해줘.
```
7. 답변을 비교해 봅니다.
